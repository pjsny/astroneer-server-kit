# astroneer-server-kit (Fly.io)

Self-hosted **Astroneer dedicated server** on [Fly.io Machines](https://fly.io/docs/machines/) ‚Äî **Docker** image on **Ubuntu 24.04** with **[WineHQ](https://wiki.winehq.org/Ubuntu) `winehq-devel` 10.6** (Noble), **GnuTLS 3.8.3** via Ubuntu **`libgnutls30t64`**, and **[DepotDownloader](https://github.com/SteamRE/DepotDownloader)** for Windows app **`728470`**. **Wine prefix** lives on the Fly volume at **`/data/winepfx`** (initialized on first boot if the image skeleton is empty). Local **`make start` / `make stop`** wrap `flyctl`. Use **BuildKit** (`DOCKER_BUILDKIT=1`) so APT layers use cache mounts and rebuilds stay faster.

## Prerequisites

- [Bun](https://bun.sh/) ‚Äî `curl -fsSL https://bun.sh/install | bash`
- [Fly CLI](https://fly.io/docs/hands-on/install-flyctl/) ‚Äî `brew install flyctl`
- Fly.io account with a payment method (Machines + volumes)

## Quick start

```bash
bun install
make setup       # wizard: Fly token, region, creates app + astroneer_server_kit_data volume, writes fly.toml + .env
make preflight
make start       # ensures dedicated IPv4 + ASTRONEER_PUBLIC_IP secret, then fly deploy ‚Äî first build + DepotDownloader fetch can take 20‚Äì40+ minutes
make ip        # show app IPv4 (same address make start uses for clients)
```

Join address: **`PublicIP:8777`**. `make start` allocates a Fly **dedicated IPv4** (if missing) and sets the **`ASTRONEER_PUBLIC_IP`** secret so `PublicIP` in `AstroServerSettings.ini` matches inbound traffic (not generic egress).

## Saves & data

World data lives on a Fly **volume** (`astroneer_server_kit_data`) mounted at **`/data`**. The game installs under **`/data/astroneer-server-kit`** (DepotDownloader). The **Wine prefix** is **`/data/winepfx`** on the volume, seeded from **`/opt/wine-prefix-skel`** on first boot. Saves symlink to **`/data/SaveGames`**. Stopping Machines keeps the volume; destroying the app may remove it ‚Äî confirm in [Fly docs](https://fly.io/docs/reference/volumes/).

## Encryption / `PublicIP`

The server‚Äôs `Engine.ini` defaults to **`net.AllowEncryption=True`**. **Clients must use the same value** in their local Astroneer `Engine.ini` or joins may fail.

Some community setups run with **encryption off** under Wine; this kit keeps **encryption on** by default. If joins fail with TLS or handshake errors, try matching **`net.AllowEncryption`** between server `Engine.ini` and clients, or temporarily disable encryption for isolation testing.

**`make start`** syncs **`ASTRONEER_PUBLIC_IP`** to your app‚Äôs leased IPv4. If joins still fail or you changed IPs manually:

```bash
fly secrets set ASTRONEER_PUBLIC_IP=YOUR_IPV4 -a "$FLY_APP_NAME"
```

Then redeploy or restart the Machine.

## Repo layout

- `Dockerfile` ‚Äî Ubuntu **24.04**, **`libgnutls30t64` (GnuTLS 3.8.3)**, WineHQ **`winehq-devel` 10.6**, DepotDownloader **3.4.0**, BuildKit APT caches, entrypoint

- `scripts/` ‚Äî Bun CLIs for setup / deploy / status
